# Unity Theory Perspective Exercises

These are companion exercises.
They should be read as interpretive or research-facing prompts layered on top of canonical ML material, not as replacements for standard definitions or proofs.

## Exercise 1. Interpretive mapping with scope control

**Taxonomy**

- `difficulty`: `intermediate`
- `type`: `analysis`
- `tags`: `interpretive-mapping`, `scope-discipline`, `representation`

Choose one canonical ML object from the list below:
- a latent variable;
- a hidden state in a sequence model; or
- a learned embedding.

Then write a short response that does all of the following:

1. states the standard ML definition or role of the object;
2. proposes one Unity Theory interpretation of the same object;
3. identifies which part of the interpretation is exact description and which part is speculative overlay; and
4. states one risk of overstating the correspondence.

## Exercise 2. Boundary-check a structural claim

Consider the claim:

> "Because a learning pipeline composes multiple transformations, Unity Theory gives the governing mathematics of the pipeline."

1. Rewrite the claim in a form that is technically defensible.
2. Identify at least two places where standard ML mathematics already does the necessary work.
3. State one narrower role that Unity Theory material can still play without replacing canonical exposition.

## Exercise 3. Companion note design

Draft a boxed companion note for a canonical lesson on optimization or representation learning.

The note should include:
1. one explicit link to the canonical concept;
2. one interpretive insight;
3. one sentence labeled as speculative if needed; and
4. one sentence explaining why the companion note is optional for core competency.
