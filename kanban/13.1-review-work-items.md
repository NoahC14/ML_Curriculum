# 13.1 Review Work Items

## Review summary

This review focused on structural completeness, curriculum consistency, and kanban readiness.

High-signal findings from the current repo state:
- `17` of `18` module `README.md` files still contain the phrase `Scaffolded placeholder`.
- The documented module contract in `shared/templates/module-template.md` is not reflected in module landing pages. `17` of `18` module READMEs do not expose the expected sections, and `13-graph-learning` only partially does so.
- Advanced-module asset coverage is uneven. `14-causality-reasoning` has notes and exercises but no notebooks or derivations. `15-ethics-safety-evaluation` has an exercise file but no substantive notes, derivations, notebooks, or project artifacts. `16-category-theory-for-ml` and `17-unity-theory-perspectives` have notes and exercises but no derivations, notebooks, or project artifacts.
- Several mid-sequence modules with substantive notebooks and notes still have no derivation artifacts: `07-deep-learning-systems`, `08-cnn-vision`, `09-sequence-models`, and `13-graph-learning`.
- The stable kanban layer is still underdeveloped. `kanban/backlog.md` and `kanban/epics.md` are starter placeholders, while `kanban/work-cards.md` still reads as a planning scaffold rather than a current execution board.
- Repo hygiene needs cleanup: committed `__pycache__` directories exist under `shared/` and `modules/06-neural-networks/src/`.

The work items below convert those findings into kanban-ready tasks.
