# Purpose
Integrate category theory and Unity Theory in a disciplined way that strengthens understanding and may generate novel research questions without destabilizing canonical ML exposition.

# Course placement rules
- Put category theory in primer mode early, especially in Module 00, using simple examples from sets, functions, vector spaces, datasets, and ML pipelines.
- Put more formal categorical consolidation in Module 16 or in advanced companion material.
- Keep Unity Theory mainly in sidebars, `unity/` notes, policy documents, or Module 17 unless the task is explicitly about scope/governance.

# Integration doctrine
There are three allowed modes:
1. structural clarification
2. interpretive correspondence
3. speculative research extension

Always identify which mode is being used.

# Structural clarification
Appropriate use:
- ML pipeline as composition of maps
- representation learning as structure-preserving transformation
- model families as classes of morphisms between spaces
- invariance or equivariance as preservation under transformations

# Interpretive correspondence
Possible correspondences:
- identity -> object / representation / state
- relation -> morphism / interaction / update
- embodiment -> concrete realization in data or architecture
- coherence -> invariance / consistency / stable generalization
- sacrifice or transformation -> constrained update or optimization dynamics

These are correspondences, not automatic equivalences.

# Speculative research extension
Allowed only when explicitly labeled. Good examples:
- proposing a new compositional learning framework;
- interpreting transfer learning categorically; and
- framing a research question around coherence-preserving representations.

# Hard constraints
- Never let interpretive language replace derivation.
- Never present Unity Theory as established ML doctrine.
- Keep canonical exposition first and companion framing second.
- Keep boxed sidebars separate from core lessons when necessary.
- When making a mapping, specify both sides of the mapping precisely.

# Output patterns
Useful outputs include:
- sidebars
- appendices
- correspondence tables
- research prompts
- formalization sketches
- comparative notes between standard and Unity-theoretic framing
